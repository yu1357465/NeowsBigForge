# NeowsBigForge 架构说明书 (Hybrid 极速自动化版)
**杀戮尖塔2 (Slay the Spire 2) 定性构筑沙盘系统**

## 1. 核心设计哲学 (Design Philosophy)

在 V2.0 的基础上，新版架构引入了“零干扰”进化：

* **定性优于定量 (Qualitative > Quantitative)：** 继续坚持端口平衡理论，将复杂的卡牌数值抽象为“过渡”、“终端”、“润滑”五大核心雷达指标。
* **交互消失论 (Zero-Interaction)：** 遵循“最好的交互就是没有交互”原则。通过本地守护进程实现从“玩家搬运数据”到“数据追随玩家”的跨越。
* **次元壁突破 (Sandbox Breaking)：** 利用轻量级 Python 代理作为底层“内应”，绕过浏览器对系统文件夹的物理隔离，实现 C 盘存档的毫秒级直接读取。

## 2. 系统层级与数据流转 (System Architecture)

系统架构从“单体网页”演变为“混合双轨”架构：

1.  **实战代理层 (Local Agent Layer)：** * **sts2_sync_agent.py**：由 Python 编写的轻量级后端。
    * **核心任务**：智能寻址（通过时间戳嗅探所有 Steam 账号和 Profile 下最新的存档）、提供 RESTful API (/list, /get) 给网页端。
    * **部署策略**：通过 Base64 编码嵌入 .bat 脚本，实现免环境配置的一键提权部署与开机自启任务注册。
2.  **持久化记忆层 (Hybrid Storage)：**
    * **LocalStorage**：存储沙盘状态、双轨环境参数（能量/抽牌）。
    * **IndexedDB**：作为旧版目录句柄的备用记忆库。
    * **CRDT 智能字典**：玩家自定义鉴定库，引入时间戳合并机制。
3.  **解析与渲染层 (Memory-First Engine)：**
    * 放弃 FileReader 物理读取流，改为内存 JSON 直读。数据通过 12026 端口直接喂入 Vue/JS 内存工作区。

## 3. 核心引擎解析 (Core Engines)

### 3.1 智能寻址与幽灵进程猎杀引擎
* **底层逻辑 (Mechanism)**：针对 Windows 环境下的端口霸占问题，部署脚本引入了“猎杀指令” (`taskkill`)。在启动新版引擎前，物理清除霸占端口的旧版进程，确保 API 定义（/list, /get）始终为最新版本。
* **智能嗅探**：Python 脚本利用通配符扫描 `%APPDATA%` 路径，并对所有匹配的 `current_run.save` 进行 `mtime`（修改时间）排序，永远锁定当前活跃的对局。

### 3.2 CRDT 无冲突数据合并引擎
* **机制 (Mechanism)**：在导入/导出玩家自定义卡牌鉴定库时，采用简化的 CRDT（Conflict-Free Replicated Data Type）思想。
* **合并规则**：
    * **心血保护**：若导入数据为空且本地有标签，跳过覆盖。
    * **状态优先**：若导入数据有标签且本地为空，强制覆盖。
    * **时间戳共识**：若双方均有内容，比对 `lastModified` 字段，以最后修改时间为准。

### 3.3 双轨宇宙内存隔离
* **隔离逻辑**：系统维护两套平行的环境参数（能量自由度、抽牌权重）。
    * **脑测模式 (Sandbox)**：参数与卡组绑定，随方案切换。
    * **实战模式 (Live)**：参数与全局环境绑定，不随存档切换而重置，确保玩家在实战解析时数值的稳定性。

### 3.4 自动化轮询与低耗休眠机制
* **底层实现**：前端移除所有“手动监控”开关。只要侦测到本地 Agent 在线，系统强制接管轮询（每 2000ms 一次心跳）。
* **性能优化**：引入 `visibilitychange` 监听。当玩家切回游戏对战时，沙盘网页进入“休眠态”停止轮询；当玩家切出游戏查看沙盘时，网页瞬间激活并拉取最新数据，确保 0 CPU 浪费。

## 4. UI 交互规范 (UI/UX Standards)

* **视觉对齐**：所有系统级操作（部署、重绑）均采用虚线边框或半透明设计，并向右侧/底部边缘靠拢，避免干扰核心推演区。
* **状态反馈**：引入“呼吸灯”交互。绿色亮起代表 Agent 直连成功，取代冗余的文字提示。
* **特效克制**：移除所有导致视觉跳动的体积缩放 (Scale)，复选框与卡牌交互统一采用 `translateY` 轻微浮动配合阴影辉光，提升工业级工具感。

## 5. 兼容性与安全性 (Compliance)

* **日志屏蔽补丁**：针对 `pythonw.exe` 无控制台环境，强制重写 `log_message` 函数，防止因无法打印日志导致的底层断路器崩溃。
* **编码防弹设计**：所有动态生成的脚本强制注入 UTF-8 BOM 头与 CRLF 换行符，彻底解决中文 Windows 环境下的乱码与 PowerShell 语句截断问题。
